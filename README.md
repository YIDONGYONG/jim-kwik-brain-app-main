# Jim Kwik C.O.D.E Test MVP

このプロジェクトは、Jim Kwikの「C.O.D.E」フレームワークに基づいて、ユーザーの脳タイプ（学習スタイル）を診断するWebアプリケーションです。

## ディレクトリ構造 (Feature-Sliced Design)

このプロジェクトは、コードの可観測性と保守性を高めるため、Feature-Sliced Design (FSD) の概念を取り入れたモジュラーアーキテクチャを採用しています。
他の開発者が機能を修正・追加する際は、以下の構成に従ってファイルを配置してください。

- `src/app/`
  - **役割**: アプリ全体の設定やエントリーポイントが含まれます。
  - **内容**: `App.tsx` (メインUIマウント), `main.tsx` (Reactのマウント), `index.css` (全般的なスタイル)。

- `src/assets/`
  - **役割**: 静的なメディアファイルの保管場所。
  - **内容**: 画像、アイコンなど。

- `src/entities/`
  - **役割**: ビジネスロジックの中核（データ、ドメインモデル、および型定義）。
  - **内容**: `codeData.ts` (質問データ、各タイプの解説)。テキストや質問内容を修正する場合はこのディレクトリを確認してください。

- `src/features/`
  - **役割**: ユーザーアクションや特定の機能に基づくUIパーツ群。
  - **内容**: `QuestionCard.tsx` (質問機能), `ProgressBar.tsx` (進捗バー), `AnimalFace.tsx` (結果の顔描画)。再利用可能な機能単位で作成しています。

- `src/scenes/`
  - **役割**: 画面（ページ）レベルのコンポーネント。
  - **内容**: `ResultReport.tsx` (診断結果画面)。

- `src/shared/`
  - **役割**: 汎用的なユーティリティやヘルパー関数。ドメインに依存しません。
  - **内容**: `utils.ts` (CSSクラスの結合関数など)。

## 機能の追加・修正ガイドライン

1. **質問文や動物タイプのデータを修正・追加したい場合**: `src/entities/codeData.ts` を編集してください。
2. **診断結果のUIやシェアリング機能を改修したい場合**: `src/scenes/ResultReport.tsx` を編集してください。
3. **新しい機能コンポーネントを追加する場合**: 関連するドメイン機能なら `src/features/`、全域で使う汎用UIなら `src/shared/` に追加します。# jim-kwik-brain-app-main
